HTML: Hypertext Markup Language

  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user
  • Add semantics to help with the acessability of your webpage and make your HTML easier to read by other devs and the browser
  • Child elements are elements that are nested inside other elements
  • Parent elements hold all the child elements
  • An element refers to the entire element from opening tag to closing tag, whereas a tag refers only to what appears inside the angle bracket
  • Use heading tags in order from h1 to h6 and only use h1 once on each page
  • Footers usually contain the author, copyright, contact, sitemap, and navigation
  • Section elements are a semantic element that define a section in a document that contains thematic content
  • target="blank": opens a href into a new tab

CSS: Cascading Style Sheets

  • Ways to style a webpage using CSS:
    • Inline CSS
    • Internal CSS style sheet
    • External CSS style sheet

  • Inline CSS:
    • Used to style an element directly in the HTML file
    • Apply to a specific HTML tag, using a style attribute with a CSS rule to style a particular page element only
    • Useful for quick and permanent changes
    • Less flexible because each inline style must be edited individually whenever you make a design change

  • Internal Style Sheet:
    • Contains CSS rules for the webpage in the head section of the HTML file
    • The rules only apply to that page. Configure CSS classes and IDs to style multiple elements in the page code

  • External Style Sheet:
    • Keeps all CSS rules in a separate file
    • Makes design changes easier
    • Linking goes in the head section

  • Cascading style sheets: reads top down, so place media queries at bottom
  • Use fluid layouts: percentage based sizing instead of px based
  • Declarations contain two components: the CSS property we want to apply and the value of the property
  • "font-family:inherit":lets child tag inherit font-family of parent tag
  • fonts load left to right until if finds one that the browser has
  • "text:transform: uppercase"
  • "text-decoration: underline/strikethrough"
  • -webkit appearance: for chrome and safari
  • -moz-appearance: for firefox

  • Display property: “display=”inline/block/inline-block/none/flex”
    • Block elements (takes up as much width as it can)(always starts on a new line): paragraph and unordered list
    • Inline elements (only takes up what it needs): anchor and image
    • Inline-block: set as inline but can apply height and width values

  • Spacing attributes:
    • "margin": indicates how much space around the outside of an element/space between content
    • "padding": adds space around the content inside an element
    • "border" can be dashed

  • The "rel" attribute specifies the relationship between the current document and the linked document resource
  • Colors can be assigned by semantic (red), hexadecimal code/number (#888888), or rgb value (rgb(250,250,250))
  • "float": align images

  • Selector: everything before css curly brackets:
    • .name: class selector
    • #name: id selector
    • name: html element
  • Selectors let us assign styles to certain elements
  • Selectors defines the element or attribute the rules/declarations will apply to
  • The * selector applies the rules to all the elements visible on the page
  • Most specific css selector takes priority

  • Pseudo Classes:
    • +, >, , ~, comma: common/popular selector combinators
      • +: second element immediately follows first
      • >: direct child element
      • space: decendant combinator
      • ~: general sibling combinator
      • comma: selects all matching nodes/selectors that have the same declarations-> div/span/all h tags
  • Pseudo element: changes the content-> h1::after/before-> "content", "color", "etc."

  • Positions
    • Static position: normal flow/default value
    • Relative position: normal flow offset by value
    • Absolute position: position relative to parent or ancestor
    • Fixed position: removed from normal document flow, relative to viewport, determined by values
  • "justify-content":align along main axis
  • "align-items": align along secondary axis
  • "order": change the flex position to a different one (side element goes under even when placed above in html)

  • Forms
    • Form types
      • button, checkbox, text, textarea, select
    • "maxlength": creates a character limit for text boxes
    • Buttons in forms will submit all inputs in the form

    Git

    • Git Flow:
        1. git checkout -b branch-name
        2. make changes
        3. git add -A
        4. git commit -m "message"
        5. git pull origin main
        6. git push origin branch-name
        7. finish pull request on github
        8. git checkout main
        9. git pull origin main

    • git status: checks what branch we are currently on
    • git touch file-name: creates a new file in the current folder/directory
    • git checkout -b branch-name: creates a new branch and switches to it

    • git add -A:
      • Adds modifications in the current working branch to the staging area
      • -A indicates we want to add all changes
      • The staging area informs Git that the changes will be added in the next commit

    • git commit -m "message":
      • Make sure to include a commit message to describe the changes implemented
      • A commit creates a snapshot of the currently staged changes

    • git pull origin main:
      • Pulls down any changes from the main branch to check if it is in sync with our local branch
      • Always do a git pull to make sure you are working on the latest version of the project on your local computer before you start to work

    • git push origin branch-name:
      • Pushes our changes to the remote branch on github
      • On github, give the pull request a descriptive title and add instructions, to the body, for reviewing team members about what changes were made/how the feature is supposed to function/how to start the application so the feature can be tested

    JavaScript

    • Variable: a named container that alows us to store data in our code
    • Variable Types:
      • String: "" or ''
      • Boolean
      • Number
      • Array: []

    • Functions:
      • A set of instructions that tells the computer how to perform a task
      • Different than a loop or conditional because functions do not automatically execute when the file is run
      • Must be called by name in order to execute and can be called as many times as needed
      • Being able to call a function to execute it means you can control where in the control flow a ask is executed, if at all

    • Control Flow is the order in which a computer executes code in a script
    • Each line of JavaScript ends wih a semicolon
    • Linking goes below the footer but above the closing body tag
    • Index is the unique identifing number for a piece of data in an array
    • Index numbers start at zero and are sequential
